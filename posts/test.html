Abstract

This paper presents design findings from a study employing a digital version of the Delta Design Game tool, originally developed by Bucciarelli in 1991. In this work, we present a digital interface in which users play the Delta Design game online while the interface collects data on the behavior of the designer. Using our tool, players can compose, revise and verify the designs on a digital platform, while the digital tool captures every action taken in the interface. For this research, we studied N=15 engineering graduate students to compare design behaviors in groups of high-performing and low-performing designers. We used Hidden Markov Modeling techniques to discern patterns of behaviors between high-performing designers and low-performing designers. Our findings from the Hidden Markov Models show that designers proceed through four procedural latent states for the design challenge. High-performing designers typically achieve optimal performance by dividing the complex novel problems into sub-problems and defining an analogy with familiar tasks. Both efficient and inefficient designers are able to achieve high-performing scores but show different cognitive processes. Interpreted through Schraagen’s theory of design cognition, the high performing designers in general showed progressive deepening strategies, progressing systematically through subtasks, starting with structural aspects of the challenge before the parametric optimization.

Setting up remote container for a project

To use this feature we would need following files and folder :

  1. .devcontainer/devcontainer.json : Required file that is basically a config file that determines how your dev container gets built and started.
  2. .devcontainer/Dockerfile : If we need a custom image as environment.
  3. app/ : The code of the application (can vary based on your project).

Within the .devcontainer folder the devcontainer.json config file helps the extension to determine the name of the container, which image to use, extensions to install, port to expose and other configurations. Few examples,

1. Shiny Application
